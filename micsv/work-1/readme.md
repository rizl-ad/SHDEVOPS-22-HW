# Решение ДЗ к занятию «Введение в микросервисы»

Переход на микросервисную архитектуру для крупного интернет-магазина с быстро растущей нагрузкой может привести к существенным выгодам, но требует серьёзной подготовки.

### Основные выгоды:
- распределение бизнес-функции по разным независимым сервисам;
- разработка и выкладка независимая от изменений в других сервисах;
- быстрая разработка и частые выкладки;
- сокращение расходов на коммуникации;
- возможность использовать разные технологии для решения разных задач;
- повышение отказоустойчивости и масштабируемости.

### Задачи, решение которых потребуется в первую очередь:
- определить решаемые проблемы и цель перехода на микросервисную архитектуру;
- найти подход к разделению системы на сервисы;
- определиться с правилами владения сервисов;
- определить способы и подходы межсервисного взаимодействия, в том числе в части ИБ;
- определить компоненты инфраструктуры под микросервисную архитектуру;
- определить методы мониторинга и сбора логов;
- определить состав персонала разработки, развёртывания и сопровождения;
- определить необходимые финансовые вложения.
